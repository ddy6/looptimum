# Migration: `v0.2.x` to `v0.3.0`

## Metadata

- Migration id: `v0.2.x-to-v0.3.0`
- Source versions: `v0.2.x`
- Target version: `v0.3.0`
- Owner: maintainers
- Date: 2026-03-05

## Scope

- Authoritative state artifact:
  - `state/bo_state.json`
- Payload and operator outputs:
  - `suggest` payload
  - `ingest` payload (optional-but-emitted from harness)
  - `status` payload
  - `doctor` payload
  - `report` payload

Out of scope:

- Objective function logic
- Surrogate/acquisition algorithm internals

## Contract Changes

Added:

- `schema_version` field:
  - format: semver string (`<major>.<minor>.<patch>`)
  - example: `"0.3.0"`

Required:

- `schema_version` is required in `state/bo_state.json` for `v0.3.0` state.

Optional-but-emitted:

- `schema_version` in `suggest` and `ingest` payloads.
- `schema_version` in `status`, `doctor`, and `report` outputs.

## Compatibility Behavior

Legacy `v0.2.x` state:

- If `schema_version` is missing or `0.2.x`, runtime upgrades in-memory to
  `0.3.0`.
- Runtime emits a loud warning with migration pointer.
- Upgraded value is persisted on the next mutating command.

`v0.3.x` compatibility guarantee:

- Any earlier `v0.3.x` state must load transparently.
- Warn-only deprecations are allowed.
- No load failures for earlier `v0.3.x` state.

Unsupported series:

- Non-`0.3.x` future-major/minor state versions are rejected with a clear error.

## Canonical Fixtures and Tests

Canonical fixture source:

- `tests/fixtures/state_versions/`

Compatibility and upgrade-path coverage:

- `templates/tests/test_state_versions.py`

Coverage includes:

- missing-version legacy state load behavior
- explicit `0.2.x` state upgrade behavior
- transparent load for earlier `0.3.x` state
- unsupported schema-series rejection
- old-state -> current runtime -> validate/report/doctor path

## Operator Notes

Recommended migration flow:

1. Back up current `state/bo_state.json`.
2. Run a non-mutating command (for visibility), e.g. `status` or `doctor`.
3. Run next mutating command (`suggest`/`ingest`/`cancel`/`retire`/`heartbeat`/`report`).
4. Confirm persisted `state.schema_version` is `0.3.0`.

No manual schema rewrite is required for standard `v0.2.x` state files.
