# v0.2.0 Release Execution Checklist

Date: 2026-03-04
Purpose: Operational checklist for phase 8 step 72 final execution.

## Preconditions

- Working tree is clean.
- Main branch is up to date.
- Local/CI gates are green (format, lint, links, tests, smoke).
- Release owner approval is granted for tag/release publication.

## Commands

Run from repository root:

```bash
git checkout main
git pull --ff-only origin main
git tag -a v0.2.0 -m "Looptimum v0.2.0"
git push origin v0.2.0
```

## GitHub Release Steps

1. Open the new tag `v0.2.0` in GitHub Releases.
2. Use `CHANGELOG.md` (`0.2.0` section) as release notes base.
3. Include migration highlights:
   - stability guarantees location
   - smoke automation availability
   - known deprecations (`success` alias, sentinel path -> `v0.3.0` removal plan)
4. Publish release.

## Post-Publish Checks

1. Confirm tag is visible on remote.
2. Confirm release notes render correctly.
3. Confirm CI status for release commit/tag is green.
4. Update `reports/update_program_tracking_issue.md` to mark step 72 complete.
